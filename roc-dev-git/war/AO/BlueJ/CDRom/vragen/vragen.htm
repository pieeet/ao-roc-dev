Vragen hoofdstuk 1

1.       Wat is een ander woord voor object?

Instantie.


2.       Wat is een constructor?

Een constructor is een gedeelte van een klasse waarmee je een object maakt.


3.       Wat is een klasse?

Een klasse is de definitie voor objecten die je ervan kunt maken. Een klasse bestaat doorgaans uit gegevens en methoden.


4.       Hoeveel instanties kun je van een klasse maken?

Net zo veel als je wilt (en als het geheugen van je computer toelaat).


5.       Naar een instantie van Lamp kun je vier verschillende berichten sturen. Hoe reageert een lamp-object na het sturen van elk van deze berichten?

Door het bericht aan() gaat de lamp aan.
Door het bericht uit() gaat de lamp uit.
Door het bericht maakZichtbaar() wordt de lamp zichtbaar gemaakt in een apart venster.
Door het bericht isAan() geeft de lamp antwoord op de vraag of hij aan is (true als dat zo is, en anders false).


6.       Wat geeft het woord boolean aan?

Het woord boolean geeft aan dat het om een waarde gaat die alleen false of true kan zijn.


7.       Wat geeft het woord void aan?

Het woord void geeft aan dat een methode geen waarde aflevert.


8.       Wat geeft het woord int aan?

Het woord int geeft aan dat het om een geheel getal gaat.


9.       Wat is de toestand van een object?

De verzameling van de waarden die in een object zitten. Als een van de waarden van een object verandert, verandert de toestand van dat object.


10.   Wat geeft het woord private aan?

Het woord private geeft aan dat het om een privé-gegeven van een object gaat.

Vragen hoofdstuk 2

1.       Wat verstaan we onder de interface van een object?

De lijst met namen van alle berichten (methoden) die je naar dat object kunt sturen.


2.       Wat betekent "een methode aanroepen"?

Is hetzelfde als een bericht naar een object sturen.


3.       Wat is de retourwaarde van een methode?

Een waarde die een methode aflevert als je hem hebt aangeroepen. Niet alle methoden leveren een waarde af, in dat geval staat er void voor de methode.


4.       Hoe kun je in BlueJ de broncode van een klasse zien?

Door met de rechter muisknop op een klasse te klikken en Open Editor te kiezen.


5.       Wat betekent compileren?

Compileren is het laten vertalen van broncode naar code die door de computer begrepen wordt.


6.       Waarom is compileren noodzakelijk?

Omdat een computer de broncode niet begrijpt.


7.       Wat is overloading?

Het verschijnsel dat een klasse een of meer methoden met dezelfde naam heeft. De methoden moeten dan wel een verschillend aantal argumenten hebben, of hun argumenten moeten van een verschillend type zijn.

Vragen hoofdstuk 3

1.       Wat is een attribuut?

Een attribuut is een gegeven van een klasse. Omdat objecten gemaakt worden op grond van de code in een klasse beschikt elk object over dezelfde attributen als in de klasse zijn gedefinieerd.


2.       Wat plaats je in de drie gedeelten van het diagram van een klasse?

In het bovenste gedeelte de naam van de klasse, in het midden de attributen, en in het onderste de methoden.


3.       Welke van de volgende benamingen zijn geldige namen in Java?
Java
U2
AZ67
8088
H&M
get saldo
get_saldo
getSaldo

Alleen 8088, H&M  en get saldo zijn niet geldig.
De schrijfwijze van get_saldo is ongebruikelijk. Beter is getSaldo.


4.       Welk type gebruik je in Java voor een stukje tekst? Voor een geheel getal? Voor een gebroken getal?

Achtereenvolgens String, int en double.


5.       Wat betekent case-sensitive?

Dat er onderscheid gemaakt wordt tussen hoofd- en kleine letters.


6.       Wat is een constructor? Welke naam heeft een constructor altijd?

Een constructor is een soort methode die je aanroept als je een nieuw object maakt. Een constructor heeft altijd dezelfde naam als de klasse.


7.       Met welk teken geef je private aan in een klassendiagram? En met welk teken public?

private geef je met een minteken aan en public met een plusteken.


8.       Wat is een getter? En wat is een setter?

Een getter is een methode die als retourwaarde de waarde van een attribuut heeft. De naam van een dergelijke methode begint met het woord get, en wordt gevolgd door de naam van het attribuut. Bijvoorbeeld:
Als er een attribuut int lengte is, dan is de bijbehorende getter:
public int getLengte()
Een setter is een methode waarmee je de waarde van een attribuut kunt veranderen. De naam van een dergelijke methode begint met het woord get, en wordt gevolgd door de naam van het attribuut. Een setter heeft een argument van hetzelfde type als het attribuut.
Bijvoorbeeld:
Als er een attribuut int lengte is, dan is de bijbehorende setter:
public void setLengte( int lengte )

Vragen hoofdstuk 4

 

1.       Wat is een associatie?

Een associatie is een relatie tussen klassen, dat wil zeggen dat de instanties van die klassen op een of andere manier met elkaar te maken hebben.


2.       Wat is een referentie?

Een referentie is een naam die verwijst naar een object.


3.       Welk teken gebruik je voor de concatenatie van strings?

Het plusteken.


4.       Hoe ziet de uitvoer van het volgende stukje broncode er precies uit?

String naam1 = "Ana";
String naam2 = "Bea";
int leeftijd = 18;

System.out.print( naam1 + " is " );
System.out.println( leeftijd );
System.out.println( naam2 + " is " + (leeftijd + 10 ) );


De uitvoer:

Ana is 18
Bea is 28


5.       Wat is een test fixture? Waarom kan dat handig zijn?

Een test fixture is een vaste verzameling objecten die je na het wijzigen van de broncode steeds automatisch opnieuw kunt laten maken. Dat is handig omdat je dan de objecten niet steeds met de hand hoeft te maken.

Vragen hoofdstuk 5

1.       Wat is een collectie? Wanneer gebruik je een collectie? Welke klasse kun je in Java voor een collectie gebruiken?
Een collectie is een object waaraan je net zoveel objecten kunt toevoegen als je wilt.
Een collectie gebruik je als je veel (of een onbekend aantal) objecten wilt kunnen opbergen.
Voor een collectie kun je de klasse ArrayList gebruiken.


2.       Waarom mag je niet definiëren
ArrayList<double> lijst;
maar wel
ArrayList<Double> lijst;

In een ArrayList kun je alleen objecten opbergen. Getallen van het type double zijn geen objecten. Getallen van het type Double zijn dat wel.

Vragen hoofdstuk 7

1.       Wat is Swing?

Swing is de naam van een library (een verzameling klassen) voor het maken van een gebruikersinterface.


2.       Over welke methode moet een applicatie beschikken?

In elk geval over een statische methode met de naam main().


3.       Wat is een statische methode?

Een statische methode is een methode waar het woord static voor staat. Een statische methode kun je aanroepen zonder dat je een instantie van de klasse gemaakt hebt.


4.       Wat is een gebruikersinterface?

Een gebruikersinterface is het geheel van knoppen, invoervakken, et cetera via welke de gebruiker en een programma met elkaar communiceren.


5.       Wat is een component?

Een van de onderdelen van een gebruikersinterface, bijvoorbeeld een knop, een label of een tekstvak.


6.       Wat is het verschil tussen een applicatie en een applet?

Een applicatie is een zelfstandig programma. Een applet is een Java-programma dat in een webpagina draait.


7.       Wat is een event? Wat is event handling?

Een event (gebeurtenis) is een actie van de gebruiker die invloed heeft op een programma, bijvoorbeeld het klikken op een knop, of het invoeren van tekst in een tekstvak.
Event handling is het afhandelen van een event, dat wil zeggen: de reactie van het programma op de actie van de gebruiker.
